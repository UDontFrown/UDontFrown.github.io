值传递和引用传递

值传递和引用传递
直接上代码：

public class Test { <br> public static void main(String[] args) { <br> int i1 = 10; <br> int i2 = 5; <br> change(i1, i2); <br> System.out.println(i1); <br> System.out.println(i2); <br> } <br> public static void change(int i1, int i2) { <br> int temp = i1; <br> i1 = i2; <br> i2 = temp; <br> } <br> }

结果如下：

会发现i1 和 i2的值并没有交换，为什么呢，代码在逻辑上看并没有问题啊？

我们再来看一段代码：

public class Test { <br> public static void main(String[] args) { <br> int[] a = {1, 2}; <br> change(a); <br> System.out.println(a[0]); <br> System.out.println(a[1]); <br> } <br> public static void change(int[] a) { <br> int temp = a[0]; <br> a[0] = a[1]; <br> a[1] = temp; <br> } <br> }

结果如下：

这下发现两个值交换了？为什么？
这就是Java中的值传递和引用传递问题，究其原因，我们来看看内存中的执行过程


值传递

进入方法后，栈中会出现两个i1, i2的副本，也可以不叫i1, i2，方法中对值得修改其实是对副本得修改，至于栈底的原值并没有被修改，所以输出后两个值没有被交换。


引用传递
实际上引用传递传递的是引用值并将内存地址中的值修改了
这段代码的流程：
1、堆内存中创建一块内存空间，存放1和2，将栈中a[0]和a[1]指向它
2、将a[0]和a[1]拷贝一份，继续指向堆内存
3、创建一个temp变量存放a[0]指向的内存地址的值也就是1
4、用a[1]指向的内存地址的值将a[0]指向内存地址的值覆盖，现在a[0]和a[1]都是2
5、将temp的值1赋给a[1]指向的内存地址，现在a[1] = 1。
一番操作下来内存中的值被修改了，但是原a[0]和a[1]还是指向原本的地址，所以a[0]指向的就是2，a[1]指向的就是1。

区分值传递和引用传递的关键就是看数据类型，基本数据类型就是值传递，引用数据类型就是引用传递，但有个特例，也就是String类型。
String类型是引用类型，按理来说应该遵守引用传递的，但我们来看代码：

public class Test { <br> public static void main(String[] args) { <br> String a = "aaa"; <br> String b = "bbb"; <br> change(a, b); <br> System.out.println(a); <br> System.out.println(b); <br> } <br> public static void change(String a , String b) { <br> String temp = a; <br> a = b; <br> b = temp; <br> } <br> }

然后发现它还是没有变化，为什么？我们再来分析一下：

因为字符串是常量，所以方法里修改它们的值只是改变了副本的引用地址，并没有影响到原数据的指向，所以String类型的传参并不会改变原数据。

总结：基本类型遵守值传递，引用类型遵守引用传递，唯一特例是String类型，引用类型却遵守值传递。
